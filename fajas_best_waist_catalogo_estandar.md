# 13. Seguridad

## 13.1 Supabase

- Activar RLS en todas las tablas expuestas.
- Permitir lectura pública solo de productos publicados.
- Restringir escritura a usuarios autorizados.
- Restringir Storage.
- Nunca exponer service_role.
- Validar permisos desde servidor.
- Validar permisos desde base de datos.
- Limitar tamaño de archivos.
- Limitar tipos de archivo.
- Implementar CAPTCHA en formularios públicos.
- Aplicar límites de frecuencia.

## 13.2 Cloudflare

- HTTPS obligatorio.
- SSL/TLS Full Strict.
- DNSSEC.
- Protección de tráfico.
- Turnstile.
- Reglas de seguridad.
- Redirección HTTP a HTTPS.
- Protección del acceso administrativo cuando sea necesario.

## 13.3 Vercel

- Variables de entorno separadas.
- Variables sensibles protegidas.
- Preview separado de producción.
- Protección de despliegues.
- Control de acceso.
- Revisión de dominios.
- No exponer secretos.

## 13.4 GitHub

- Repositorio privado.
- Rama main protegida.
- Pull Request obligatorio.
- Pruebas obligatorias.
- Bloquear force push.
- Activar Dependabot.
- Revisar dependencias.
- No guardar credenciales.

## 13.5 Zoho Mail

Configurar:

- MX.
- SPF.
- DKIM.
- DMARC.
- MFA.
- Contraseñas seguras.
- Alias corporativos.
- Recuperación segura.
